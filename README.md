## 📑 Аналитика

Каждая игра начинается с вызова метода `POST /new` с указанием размера игрового поля `width` и `height`, а также количества мин `mines_count` на нём. Для тестовой реализации ограничиваемся разумными размерами игрового поля: ширина и высота не менее 2 и не более 50, количество мин не более `width * height - 1` (всегда должна быть хотя бы одна свободная ячейка).

Далее игроку предлагается в созданной игре (идентификация игры по полученному в ответ `game_id`) делать ходы методом `POST /turn`, указывая координаты ячейки, которую игрок хочет открыть. В запросе `row` - номер ряда, начиная с нуля, и `col` - номер колонки, начиная с нуля.

В ответ на оба метода приходят данные о самой игре: уникальный идентификатор игры `game_id`, размеры поля `width` и `height` и количество мин `mines_count`, указанные при создании игры, а также параметр `completed`, указывающий, завершена ли текущая игра. Данные об игровом поле `field` возвращаются в виде двумерного массива символов (height строк, в каждой по width элементов). Игровое поле состоит из следующих символов: пустые строки `" "` (пробелы) означают неоткрытые ячейки поля, поля с цифрами от `"0"` до `"8"` означают открытые ячейки, где цифры указывают, сколько мин расположено в непосредственной близости от текущей ячейки.

Игра завершается (возвращается `completed: true`) в одном из двух случаев:

### 🟩 Победа:

1) пользователь открыл все ячейки, не занятые минами;
2) все ячейки с минами отмечаются символами "открытых" мин `"M"` (латинская заглавная "эм").

### 🟥 Проигрыш:

1) пользователь указал на ячейку, где была установлена мина;
2) все ячейки с минами при этом отмечаются символом "взорванной" мины `"X"` (латинская заглавная "икс");
3) а также должно полностью открыться игровое поле: в ячейках без мин должно отображаться количество мин рядом с ними.

Если в процессе игры пользователь открывает ячейку, рядом с которой нет ни одной мины (то есть ячейка со значением `"0"`), должны рекурсивно "открыться" все смежные ячейки, рядом с которыми также нет ни одной мины, а также все смежные с ними "числовые" ячейки, рядом с которыми мины есть, с указанием их количества. Фактически открывается область смежных "нулевых" ячеек с "числовыми" границами.

Не допускается после завершения игры делать новые ходы, а также повторно проверять уже проверенную ячейку. Эти, а также иные ошибочные ситуации, такие, как неуспешная проверка корректности входных данных, должны возвращать ошибку с кодом 400 Bad Request с текстовым описание ошибки в error. Формат ошибки - человекочитаемый текст, именно он будет отображаться в интерфейсе.

Важно!
Фронтенд самостоятельно не реализует никакую дополнительную логику, а лишь отображает то, что ему предоставляет бэкенд.
Фронтенд осуществляет лишь формальную проверку полученных от бэкенда данных:

Наличие только допустимых символов в описании игрового поля (пробел `" "`, строки с цифрами от `"0"` до `"8"`, а также открытые `"M"` или взорванные `"X"` мины) невозможность отображения на игровом поле `"X"` или `"M"` для незавершённой игры (`completed: false`) невозможность одновременного отображения `"X"` и `"M"` или же отсутствие этих символов, а также отображение неоткрытых ячеек `" "` для завершённой игры (completed: true). То есть, если передать в ячейке игрового поля `"M"` или `"X"`, но при этом не сообщать об окончании игры (`completed: false`), то на поле действительно отобразится "открытая" или "взорванная" мина. Фронтенд игру при этом не остановит, появится лишь предупреждение. Или же, если было указано, что игра была закончена (`completed: true`), фронтенд не будет ограничивать последующие нажатия на поле в интерфейсе, бэкенд должен сам возвращать ошибку при попытке дальнейших действий в уже завершённой игре. Именно бэкенд указывает фронтенду, что именно и как отображать на игровом поле.

## 🏗 Сборка и запуск

### Сборка

```bash
mvn clean install -U
```

### Запуск

```bash
java -jar ./target/minesweeper-back-*.jar
```

Если необходимо изменить количество сохраненных сессий:
```bash
java -jar ./target/minesweeper-back-*.jar --cache.session.user.size=50
```

> В Windows `CMD` или `PowerShell` нужно прописывать полное название файла.
>
> ```bash
> java -jar .\target\minesweeper-back-1.0.0.jar
> ```
